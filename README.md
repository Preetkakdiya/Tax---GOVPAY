# 🏛️ GovPay+

**A Next-Generation Centralized Payroll, Taxation, and Corporate Compliance Gateway**

GovPay+ is a robust, highly-secure **MERN stack** application engineered to automate the ingestion, calculation, and distribution of nationwide corporate salaries. It enforces absolute cryptographically-secure taxation laws uniformly, ensuring private entities remain perfectly compliant, while providing unparalleled transparency directly to the national workforce.

---

## ⚡ Core Application Flow

1. **Corporate Submission**: A Company logs in, bulk-registers their employee roster via PAN authentication, and systematically declares the Annual CTC for each employee.
2. **Algorithmic Withholding**: The GovPay+ backend intercepts these payroll requests and dynamically pulls the live national Tax Brackets from the `TaxConfig` vault.
3. **Admin Arbitration**: The Regional Admin (Government Body) reviews the macro-level transactions. Upon approval, Net Salaries are calculated and Audit Logs solidify the action.
4. **Real-Time Disbursement**: Utilizing WebSockets (`Socket.io`), the moment a Salary is approved, the connected Employees are instantly notified on their live dashboards, enabling them to instantly download cryptographic PDF Payslips.

---

## 🔥 Professional Features Included

- **Dynamic Bracket Tax Engine**: Admins have absolute control over global tax rates, Education Cess percentages, and Surcharges. Calculations automatically apply geographically.
- **Bi-Directional WebSockets**: Socket.io provides real-time "0-refresh" UI syncing across the Admin, Company, and Employee terminals.
- **System Audit Ledgers**: Every critical mutation is permanently logged into an `AuditLog` mapping, proving what user modified what entity and when.
- **PAN-Verified Password Recovery**: Bypasses typical email vulnerability by forcing multifactor recovery using absolute Employee PAN parameters integrated over the API.
- **High-Fidelity Recharts Visualization**: Complex area and timeline graphs dynamically render macro corporate footprints, measuring exactly how much tax has been siphoned per-company in real time.
- **Role-Based Announcement Matrix**: Admins can dispatch emergency Broadcasts that target selective scopes (Only Companies, or Only Employees) rendering visually as notifications immediately.

## 🗄️ Database Schemas (MongoDB)

1. **`User` Table**: Core identity. Secures `email`, `password`, and determines `role` (`admin`, `company`, `employee`).
2. **`Company` Table**: Corporate identifiers, linked robustly to a `User` identity for RBAC traversal.
3. **`Employee` Table**: Holds critical demographic hooks (`pan`, `ctc`, `designation`, `bank_account`, `ifsc_code`).
4. **`SalaryRecord` Table**: The financial artifact. Stores exact monolithic mappings of `ctc`, derived `monthly_salary`, newly structured `annual_tax`, `monthly_tds`, precise `net_salary`, and tracking `status` (`pending`, `approved`, `rejected`).
5. **`TaxConfig` Table**: A global configuration engine storing arrays of objects `{ limit, rate }` to dynamically build tax equations.
6. **`AuditLog` Table**: Stores timestamps, the `action` string, and precisely maps back to the `User` initiator.
7. **`Complaint` Table**: Bi-directional support tickets filed by Employees, carrying `status` and `fine_amount` variables.
8. **`Announcement` Table**: Administrative Broadcasts holding a `target_audience` string allowing exact UI destination routing.

## 🚀 Installation & Deployment Guide

### Prerequisites

- Install **Node.js** (v16.14.0 or higher)
- Install **MongoDB** locally, or have an active **MongoDB Atlas Cluster URI**.

### 1. Backend Initialization

Open a terminal and navigate to the backend engine folder:

```bash
cd backend

# Install all Node backend dependencies
npm install

# Create a secure Environment Variables file
# Put the following inside a `.env` file at the root of `backend/`:
# MONGO_URI=mongodb://127.0.0.1:27017/govpay
# JWT_SECRET=your_super_secret_key_12345
# PORT=5000

# Spin up the backend API and Application Servers
node index.js
```

_The server will boot and connect its Mongoose and Socket.io instances on `http://localhost:5000`._

### 2. Frontend Initialization

Open a secondary terminal and boot up the React application:

```bash
cd frontend

# Install all Vite + React frontend dependencies
npm install

# Launch the hot-reloading development server
npm run dev
```

_The client will boot instantly and can be accessed at `http://localhost:5173`._

### 3. Usage Context

- Create your first Admin account by hitting `Sign Up` or use the database seeder if applicable.
- Once an Admin is logged in, you can create new Corporate endpoints and instruct them to begin bulk-generating their Roster.

### 4. Zero-Vulnerability Docker Setup (Production Multi-Stage)

This project has been highly optimized to run as a single container, mounting the `Node.js` environment alongside statically served compiled frontend assets.

```bash
# Build the minimal unprivileged dockerized application container
docker build -t govpay-plus .

# Deploy safely mapping out the core API
docker run -p 5000:5000 -d govpay-plus
```

### 5. Docker Hub Registry (Push & Pull)

To securely distribute your compiled image across your infrastructure, you can upload it to Docker Hub:

```bash
# 1. Authenticate with your Docker registry
docker login

# 2. Tag the secure image with your repository name
docker tag govpay-plus:latest preetkakdiya/govpay-plus:latest

# 3. Push the artifact to the cloud
docker push preetkakdiya/govpay-plus:latest

# 4. Pull and execute on any standard production server globally
docker pull preetkakdiya/govpay-plus:latest
docker run -p 5000:5000 -d preetkakdiya/govpay-plus:latest
```
